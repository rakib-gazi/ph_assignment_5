What is the Document Object Model (DOM)?

The DOM is a W3C (World Wide Web Consortium) standard. The DOM defines a standard for accessing documents. "The W3C Document Object Model (DOM) is a platform and language-neutral interface that allows programs and scripts to dynamically access and update the content, structure, and style of a document."

The W3C DOM standard is separated into 3 different parts:

  • TCore DOM - standard model for all document types
  • XML DOM - standard model for XML documents
  • HTML DOM - standard model for HTML documents
How do you select an element from the DOM?

The document object represents our web page. If we want to access any element in an HTML page, we can always start with accessing the document object.

Below are some examples of how we can use the document object to access and manipulate HTML.

  • document.getElementById(): This method selects an element by its unique id. It's efficient for selecting single, known elements quickly.
  • document.getElementsByClassName(): This selects all elements with a specified class name. It returns a live collection of elements, useful when you need to apply changes to multiple elements.
  • document.getElementsByTagName(): This method selects elements by their tag name (e.g., div, p). It returns all elements with the specified tag.
  • document.querySelector(): This method selects the first element that matches a given CSS selector. It’s versatile and supports complex selections.
  • document.querySelectorAll(): Similar to querySelector(), this method returns all elements matching a CSS selector, providing a static collection of elements.
What is event delegation in the context of the DOM, and why is it useful?

Event Delegation is basically a pattern to handle events efficiently. Instead of adding an event listener to each and every similar element, we can add an event listener to a parent element and call an event on a particular target using the .target property of the event object.

How Event Delegation Works:

  • Event Bubbling: When an event (like a click) is triggered on a child element, it bubbles up through the DOM tree, passing through all of its parent elements until it reaches the root (document).
  • Event Listener on Parent: By placing an event listener on a parent element, you can catch events from any child element by inspecting the event.target to determine which child triggered the event.

Why Event Delegation is Useful:

  • Performance Efficiency: Reduces the need to attach separate event listeners to every child element, which can save memory and processing power, especially when handling a large number of elements.
  • Handles Dynamically Added Elements: Dynamically created elements (e.g., elements added by JavaScript) do not need their own event listeners. The parent’s listener will still catch events from these new elements.
  • Simplified Code: With one event listener for a parent, code is easier to manage and maintain, reducing redundancy and potential bugs when handling events across many elements.
How do you manipulate an element's attributes and styles using the DOM?

We can manipulate an element's attributes and styles using the DOM in various way. Some of Example are given below.

Attribute Manipulate:

Attributes are properties like src, href, alt, id, etc., that are defined within an HTML tag. Using JavaScript, we can get, set, or remove these attributes.

Common Methods for Attributes Manipulating:

  • getAttribute(): Retrieves the value of an attribute.
  • setAttribute(): Sets a new value for an attribute.
  • removeAttribute(): Removes an attribute.

Style Manipulate:

To change an element's inline styles, we can access the style property of an element. This allows you to modify CSS properties directly from JavaScript.

  • element.style.property = value: The style property is used to set or modify the inline styles of an element. In JavaScript, CSS property names are written in camelCase.
  • element.style.cssText: This property can be used to set multiple CSS styles at once in a string format.

    • For non-inline styles, we can use the getComputedStyle() method to get the final computed values of styles applied to an element, which includes styles from stylesheets and inline styles.

Using classList for Style Manipulation:

Instead of modifying the style attribute directly, the classList property is often more efficient for applying styles through CSS classes. It provides several methods to manipulate CSS classes:

  • add(className): Adds a specified class to the element.
  • remove(className): Removes a specified class from the element.
  • toggle(className): Toggles the presence of a specified class on the element.
  • contains(className): Checks whether the element has a specified class.
